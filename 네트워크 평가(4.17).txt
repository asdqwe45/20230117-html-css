네트워크

LAN < WAN < INTERNET

LAN
255*255 가 소규모 2개 부서

WAN (INTRANET)
부산지사와 서울지사 연결

INTERNET (WAN을 모아 둔것)
전세계 연결


프로토콜
 - 통신을 하는 대상이 지켜야하는 규칙
 - 다른 프로토콜을 쓰면 통신 안됨
 - sw/HW 프로토콜 둘 다 존재
 - SW프로토콜 예시) Handshake, date format, 오류 검출법
 => 3번 주고받는다.
 => 헤더가 다른면 안된다.
 => 검사


IP
 - 인터넷 프로토콜 (인터넷에서만 쓴다.)
 - 비연결성 비신뢰성
 => 비연결성 : 데이터 전달 보장 안됨
 => 비신뢰성 : 데이터 순서 보장 안됨

TCP
 - IP의 비연결성 비신뢰성 보완
 - 3 WAY HANDSHAKE
 - 데이터 전달 및 순서 보장
 - 속도 느림

UDP ( http의
 - IP의 특징 그대로
 - 비연결성 비신뢰성
 - 데이터 전달 및 순서 보장 안됨
 - 속도빠름(동영상, 음성)

CSMA/CD
 - 랜통신의 표준인 이더넷에서 사용하는 프로토콜

이더넷
 - ethernet
 - CSMA/CD - LAN 통신
 - IP가 아니라 MAC주소(물리적 주소) 사용
 - 요즘도 쓰임

이더넷 케이블
 - UTP 케이블(랜선)
 - STP (방해전파 호일 추가)
 - CAT5e, CAT6, CAT7 등 존재
 - 선이 오래되면 속도 하락

다이렉트 / 크로스
 - UTP 케이블 연결방식
 - 다이렉트 => 1:1연결 (일반적)
 - 크로스 => 섞어서 연결

스위치
 - 스위칭허브라고도 불림
 - 외부 네트워크와 연결 가능?X
 - MAC이 있는 포트에 신호전달

라우터
 - 공유기를 홈 라우터라고 함.
 - 인터넷
 - 라우터가 없으면 인터넷 사용 불가
 - 외부네트워크와 연결 가능

스위치 vs 라우터
라우터는 외부망으로 연결하는 게이트웨이 역할. 즉 인터넷 가능
스위치는 내부망에서만 사용. 인터넷 불가능

리피터
 - 통신거리가 길어지면 신호가 약해지기에 리피터를 사용해 신호세기 복원, 잡음제거
 - 요즘은 보기 힘든데, 라우터, 스위치에 내장되어 있기 때문
